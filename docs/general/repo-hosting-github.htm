Github

SSH

Repositories on Github can be accessed with the SSH protocol. The advantage is that you don't have to supply username and password to connect to the servers on Github.

You need to have SSL and a GIT client installed on your machine (which is covered elsewhere).

Instructions

  1. Generate an SSH public and private key pair (SSH key). The key is tied to your identity and can be used with multiple [remote] services.
    1. Open a console window (Windows cmd.exe, Git Bash or WSL Linux).
    2. Check for existing key files: 
      ls -al ~/.ssh
      If you find files named id_ed25519, id_ecdsa or id_rsa, proceed with the next step.
    3. Otherwise run the following command: 
      ssh-keygen -t ed25519 -C "your-email@example.com"
      Replace the example above with the email address you registered with Github.
    4. When prompted for a filename, press [enter] to accept the default filename.
    5. Add a passphrase when asked to. This add an extra layer of security. The passphrase is NOT your Github account password, it used to lock the SSH key only. To unlock the key later, type the passphrase when prompted.
    6. When done, the files are located in your home directory in the .ssh subfolder (c:\users\you\.ssh\ on Windows or ~/.ssh on *UX machines.
  2. Add the key to the SSH Agent on your local machine
    1. On Windows, make sure the ssh-agent is enabled. Run services.msc, locate the "SSH Authentication Agent", and check in the properties that the start type is "manual" or "automatic".
    2. If necessary start the SSH Agent with the command 
      sc start ssh-agent
    3. Add the key to the SSH Agent with 
      ssh-add ~/.ssh/id_ed25519
  3. Register the publick key with Githug
    1. Navigate your web browser to https://github.com/your-name and log-in.
    2. Navigate to github.com/settings/keys.
    3. Click on the New SSH key button.
    4. Supply a title for the key, something like "My personal SSH key".
    5. Paste the contents of ~/.ssh/id_ed25519.pub into the "Key" textbox.
    6. Click on the Add SSH key button.
    7. If prompted, enter your Github account password.
  4. Test SSH authentication
    1. Open a console, and execute the following command: 
      ssh -T git@github.com
    2. When asked, verify the remote host fingerprint, then type yes to accept the remote host.
    3. If everything is okay, you should see the following message: 
      Hi username! You've successfully authenticated, but GitHub does not provide shell access.
      If not, go troubleshooting.
